# 15. Cursor 사용 지침

아래 지침은 이 템플릿을 실제로 채울 때 Cursor 또는 다른 AI 코딩 도구가 따라야 할 작업 방식이다.

## 기본 원칙
- 먼저 현재 코드베이스를 분석한다.
- 바로 구현하지 말고 이 템플릿을 먼저 채운다.
- `dev_support`를 최종 제품 본체로 본다.
- `ai_office`는 참조 원본으로 분석한다.
- 기존 `dev_support` 기능은 깨지지 않게 유지한다.
- Discord 결합 로직은 직접 이식하지 말고 분리/격리/치환 여부를 먼저 판단한다.

## 분석 시 반드시 포함할 것
- 재사용 가능한 순수 로직 후보
- 서버 전용으로 이동해야 하는 로직
- Discord 결합 로직 목록
- UI 대체가 필요한 흐름
- 즉시 이관 가능한 유틸/서비스
- 예상 변경 파일 목록

## 구현 전 반드시 정리할 것
- 폴더 구조 제안
- API route 설계안
- 상태 관리 방식
- 보안상 클라이언트/서버 분리 포인트
- 점진적 마이그레이션 전략
- 검증 계획

- 민감정보의 클라이언트 노출
- 검증 없는 완료 선언
- README/docs 갱신 누락
