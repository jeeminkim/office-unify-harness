# Cursor Prompt - Analysis Only

당신은 Next.js/React 프런트엔드 아키텍트이자 Node.js/TypeScript 백엔드 리팩토링 전문가다.

현재 작업은 `ai_office`와 `dev_support`를 하나의 통합 웹 서비스로 재구성하는 프로젝트이며,
최종 방향은 **`dev_support` 중심 통합 + `ai_office` 서버 로직 흡수**다.

현재 작업 루트에는 아래 두 저장소가 함께 있다고 가정한다.

- `ai_office/` : 기존 Discord 기반 AI Office 서비스 원본 참조용
- `dev_support/` : 현재 React/Next.js 기반 업무 도우미 앱, 통합의 최종 타깃

중요:
- 지금은 **코드 수정하지 마라**
- 지금은 **분석만 수행하라**
- 파일 생성, 삭제, 리팩토링, 이동, 코드 패치 모두 하지 마라
- 설명, 분류, 계획 초안, 변경 예상까지만 수행하라

---

## 반드시 따를 프로젝트 원칙

1. 최종 제품 본체는 `dev_support`다.
2. `ai_office`는 기능 원천이자 참조 원본이다.
3. 기존 `dev_support` 기능은 깨지면 안 된다.
4. Discord 결합 로직은 웹에 직접 이식하지 않는다.
5. 순수 비즈니스 로직은 가능한 한 재사용 후보로 본다.
6. 민감정보, API Key, Supabase service role key 등은 서버 전용으로 유지해야 한다.
7. 통합은 항상 분석 → 설계 → 구현 → 검증 → 문서화 순서로 진행한다.
8. 지금 단계에서는 구현이 아니라 분석과 분류가 목표다.

## 이번 분석의 목표

아래를 중심으로 현재 코드베이스를 분석하라.

### 1. `dev_support` 분석
- 현재 핵심 기능은 무엇인가
- 유지해야 하는 기존 기능은 무엇인가
- 현재 구조의 강점은 무엇인가
- 통합 시 제약이 될 수 있는 부분은 무엇인가

### 2. `ai_office` 분석
- 핵심 기능은 무엇인가
- 어떤 기능이 웹 전환 가치가 높은가
- 어떤 부분이 Discord에 강하게 결합되어 있는가
- 어떤 로직이 순수 로직으로 추출 가능해 보이는가

### 3. 통합 관점 분류
아래 네 범주로 분류하라.

#### A. 순수 로직
- 재사용 가능한 핵심 로직
- 웹/Discord와 무관하게 추출 가능한 로직
- 서버 계층으로 흡수 후보가 되는 로직

#### B. 서버 전용 로직
- 외부 API 호출
- LLM provider 연동
- Supabase 접근
- 민감정보를 다루는 처리
- 서버 계층으로 이동해야 하는 로직

#### C. Discord 결합 로직
- interaction 처리
- panel restore
- ephemeral reply
- customId 라우팅
- Discord message formatting
- 웹에 직접 이식하면 안 되는 로직

#### D. 운영 전용 로직
- 운영/제어 패널
- 상태 확인
- 관리자 액션
- 재기동/알림/운영 흐름
- 웹으로 읽기 중심 전환 가능성이 있는 로직

## 특히 보고 싶은 내용

아래 항목을 반드시 포함하라.

1. `ai_office` 내부에서 **웹으로 옮기기 쉬운 순수 로직 후보 목록**
2. **절대 바로 옮기면 안 되는 Discord 결합 코드 목록**
3. `dev_support`를 통합 앱으로 확장할 때의 **예상 폴더 구조 방향**
4. 상태 관리 방식에 대한 **초기 제안**
5. API route 설계 시 고려해야 할 **핵심 엔드포인트 후보**
6. 클라이언트/서버 분리 관점에서의 **보안 주의점**
7. 점진적 마이그레이션 관점에서의 **1차 우선순위**

## 분석 방식

다음 순서로 분석하라.

1. 관련 디렉토리와 핵심 파일을 먼저 식별하라
2. `dev_support`와 `ai_office`의 구조 차이를 요약하라
3. 재사용 가능한 순수 로직 후보를 분류하라
4. Discord 결합 로직을 분리해서 정리하라
5. 웹 전환이 쉬운 흐름과 어려운 흐름을 구분하라
6. 1차 MVP 통합 범위에 적합한 기능을 제안하라
7. 실제 구현 시 예상되는 변경 파일 범주를 예측하라

- 이유

#### C. Discord 결합 로직
- 목록
- 이유

#### D. 운영 전용 로직
- 목록
- 이유

### 5. 통합 관점 제안
