# Office Unify Report Template

## 문서 목적
이 문서는 `office-unify` 통합 프로젝트에서
분석, 설계, 구현, 검증 결과를 일관된 형식으로 보고하기 위한 표준 템플릿이다.

이 템플릿의 목적은 다음과 같다.

- 작업 결과 보고 형식 통일
- 변경 범위와 이유 명확화
- 검증 여부와 미검증 영역 구분
- 리스크와 다음 단계 가시화
- Cursor 또는 다른 AI 도구의 결과물 품질 안정화

이 템플릿은 아래 유형의 작업 결과에 사용할 수 있다.

- 분석 결과 보고

- 구현 결과 보고
- 검증 결과 보고
- 문서화 결과 보고

# 13. 작성 원칙

이 템플릿을 사용할 때는 아래 원칙을 반드시 지킨다.

1. 검증한 것과 검증하지 못한 것을 구분한다.
2. 사실과 추정을 구분한다.
3. Done과 Partial Done을 혼동하지 않는다.
4. 남은 리스크를 숨기지 않는다.
5. 변경 파일과 변경 이유를 함께 적는다.
6. 문서 반영 필요 여부를 빠뜨리지 않는다.
7. 기존 기능 회귀 가능성을 반드시 점검한다.

# 14. 사용 예시

이 템플릿은 아래처럼 사용할 수 있다.

## 예시 1. 분석 결과 보고
- 작업 유형: 분석
- 변경 파일 목록: 없음
- 상태 판정: Analysis Only

## 예시 2. 설계 결과 보고
- 작업 유형: 설계
- 변경 파일 목록: 계획 문서 또는 설계 문서
- 상태 판정: Design Only

## 예시 3. 1차 구현 결과 보고
- 작업 유형: 구현
- 신규 라우트/메뉴 추가
- 기본 검증 수행
- 상태 판정: Partial Done 또는 Done

## 예시 4. 검증/문서화 결과 보고
- 작업 유형: 검증/문서화
- lint/build/typecheck 정리
- README/docs 반영
- 상태 판정: Done 또는 Partial Done

# 15. 결론

이 템플릿은 단순한 형식 문서가 아니다.

이 문서는
- 작업 결과를 더 읽기 쉽게 만들고
- 누락을 줄이며
- 검증과 리스크를 드러내고
- 하네스 엔지니어링의 결과 보고 품질을 안정화하기 위한
표준 보고 도구다.

`office-unify` 관련 주요 작업은 가능하면 이 템플릿 형식에 맞춰 정리한다.
